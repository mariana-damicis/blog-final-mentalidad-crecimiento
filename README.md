# Aprendiendo normalización de datos: una experiencia durante mi primer proyecto con SQL

## Contexto

Durante la materia de Data Analytics tuve mi primer acercamiento al diseño de bases de datos relacionales utilizando SQL. Como parte de una actividad práctica, debíamos normalizar un conjunto de datos para construir un modelo de base de datos correctamente organizado y evitar la redundancia de información.

## Problema

Al principio comprendía el concepto de normalización de forma teórica, pero me resultaba muy difícil aplicarlo. Mi principal duda era identificar qué información debía normalizar y cuál podía permanecer en la misma tabla sin generar redundancia. También me costaba reconocer cuándo un dato debía convertirse en una nueva entidad y cómo definir correctamente las claves primarias y las claves foráneas para relacionar las tablas.

## Acciones

Para resolver estas dudas consulté al profesor y repasé nuevamente el ejercicio paso a paso. En lugar de concentrarme únicamente en las reglas de normalización, empecé a analizar la función de cada dato dentro de la base de datos y las relaciones que existían entre las distintas tablas.

Al finalizar la actividad hice una breve reflexión sobre mi proceso de aprendizaje (post-mortem). Comprendí que mi error había sido intentar aplicar las reglas de memoria, sin detenerme a entender la lógica del modelo de datos. Cuando empecé a pensar en cómo se relacionaba la información, el ejercicio comenzó a tener mucho más sentido.

## Aprendizajes

Esta experiencia me permitió comprender que la normalización no consiste solamente en dividir información en distintas tablas, sino en organizar los datos de manera que se reduzca la redundancia y se mantenga la integridad de la información.

Además, aprendí la importancia de identificar correctamente las claves primarias y las claves foráneas, ya que son las que permiten establecer relaciones consistentes entre las tablas y construir una base de datos bien diseñada.

## Evidencia de control de versiones

Este blog fue desarrollado utilizando GitHub. Durante su elaboración se realizaron distintos commits para registrar las modificaciones realizadas y mantener un historial de cambios del proyecto.

## Reflexión sobre feedback radicalmente sincero

El feedback del profesor fue fundamental para destrabar mi aprendizaje. En lugar de limitarse a indicar que la solución era incorrecta, me hizo reflexionar sobre la función de cada dato y las relaciones entre las tablas. Ese enfoque me permitió comprender el problema desde otra perspectiva y entender que pedir ayuda y aceptar comentarios constructivos forma parte del proceso de aprendizaje. Esta experiencia fortaleció mi mentalidad de crecimiento y me motivó a enfrentar nuevos desafíos con mayor confianza.

## Conclusión

Esta experiencia me permitió comprender que aprender tecnologías nuevas requiere tiempo, práctica y disposición para recibir feedback. Más allá del resultado técnico, considero que desarrollar una mentalidad de crecimiento es fundamental para seguir avanzando en mi formación como futura profesional en Data Analytics.
